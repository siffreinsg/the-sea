## Cross-cutting decisions

### Auth — Authelia, not Authentik
- Chosen over Authentik because its config is a **YAML file** → SOPS-encrypt → in
  git, fitting the "everything in git, drop the age key, redeploy" DR model.
  Authentik's config lives in Postgres and can't. Also 1 container vs 4.
- Storage: **SQLite** (local file). Sessions in-memory (no redis) — acceptable for
  a homelab; a restart just forces re-login.
- Config + `users_database.yml` are SOPS-encrypted, committed.
- Integration: **Caddy forward-auth**. Apps with no real login of their own
  (code-server, Actual, Bazarr, Dawarich, your_spotify, Homepage) sit behind it.
  Wizarr (public invite links) and your_spotify (OAuth callback) keep those paths
  public — forward-auth scoped to admin routes only.

### Bazarr media access
- Bazarr runs on GM but the media + arr stack live on The Thousand Sunny (can't run
  a 2nd Bazarr there). A prior rclone mount was abandoned only because it went
  stale — a maintenance problem, not an incompatibility. FUSE confirmed working in a
  container on GM's OpenVZ kernel (`FUSE_OK` test passed).
- Design: **Komodo-managed `rclone` SFTP mount** → Sunny over the mesh
  (`After=tailscaled` semantics), bind-mounted rw into Bazarr. Komodo's image-update
  polling keeps rclone current — fixes the root cause.
- **Audio-sync / Whisper: off** (CPU cost). Provider subtitle downloads only, so
  Bazarr needs the media tree visible to write `.srt`, not to stream whole videos.
- **Task 0 — verify GM→Sunny inbound SSH reachability.** The load-bearing, untested
  assumption: Sunny runs **userspace** tailscale and today only makes *outbound*
  mesh connections. SFTP needs GM to connect *into* Sunny's SSH — userspace tailscale
  does not bridge inbound tailnet→host ports automatically. Test on GM:
  `nc -vz <sunny-mesh-ip> 22`. If it refuses, expose it with `tailscale serve
  --tcp=22 …` on Sunny (or reconsider Bazarr-on-GM entirely). No fallback below
  covers this failure — both fallbacks assume the SSH path works.
- **Path mapping (make-or-break).** Sonarr/Radarr report *their* paths; Bazarr sees
  the mount at a different path. Bazarr's path-mapping must translate the two or subs
  land nowhere.
- Secondary risk: mount propagation into the Bazarr container on OpenVZ.
  Fallback: host `systemd` rclone mount (`After=tailscaled`), bind normally.

### Backups
Per operating rules, every stateful app gets a dump + Backrest plan entry:
- Critical plan: Authelia config/db, Actual, n8n exports, Dawarich (pg_dump).
- Bulk plan: your_spotify (mongodump), Bazarr config, cold config dirs.
